## 🎯 一、微服务架构核心

### 1. API网关（Spring Cloud Gateway）

#### 核心配置文件
- **`mall-gateway/src/main/resources/application.yml`** ⭐⭐⭐
  - 路由配置（routes）
  - 服务发现配置（discovery.locator）
  - Sa-Token配置
  - Knife4j文档聚合配置

#### 核心类
- **`mall-gateway/src/main/java/com/macro/mall/config/SaTokenConfig.java`** ⭐⭐⭐
  - **技术要点**：
    - SaReactorFilter：响应式网关过滤器
    - 路由鉴权逻辑：`/mall-portal/**` 会员认证，`/mall-admin/**` 管理员认证
    - 动态权限校验：从Redis读取路径-资源映射，使用AntPathMatcher匹配
    - 异常处理：统一处理NotLoginException、NotPermissionException
  - **关键代码**：
    ```java
    // 路径匹配器
    PathMatcher pathMatcher = new AntPathMatcher();
    // 从Redis获取权限规则
    Map<Object, Object> pathResourceMap = redisTemplate.opsForHash().entries(AuthConstant.PATH_RESOURCE_MAP);
    ```

- **`mall-gateway/src/main/java/com/macro/mall/config/IgnoreUrlsConfig.java`**
  - 白名单URL配置（通过`@ConfigurationProperties`绑定）

- **`mall-gateway/src/main/java/com/macro/mall/config/GlobalCorsConfig.java`**
  - 跨域配置（CorsWebFilter）

## 🔴 四、Redis核心

### 1. Redis基础配置

    // JSON序列化器
    Jackson2JsonRedisSerializer<Object> serializer = new Jackson2JsonRedisSerializer<>(Object.class);
    // 支持类型识别
    objectMapper.activateDefaultTyping(LaissezFaireSubTypeValidator.instance, ObjectMapper.DefaultTyping.NON_FINAL);

## 📨 五、消息队列核心（RabbitMQ）

### 1. 队列配置
- **`mall-portal/src/main/java/com/macro/mall/portal/config/RabbitMqConfig.java`** ⭐⭐⭐
  - **技术要点**：
    - 延迟队列实现：使用死信队列（DLX）机制
    - 订单超时取消场景
  - **关键配置**：
    ```java
    // 延迟队列（死信队列）
    QueueBuilder.durable(QueueEnum.QUEUE_TTL_ORDER_CANCEL.getName())
        .withArgument("x-dead-letter-exchange", QueueEnum.QUEUE_ORDER_CANCEL.getExchange())
        .withArgument("x-dead-letter-routing-key", QueueEnum.QUEUE_ORDER_CANCEL.getRouteKey())
    ```
  - **工作原理**：
    1. 消息发送到延迟队列（TTL队列）
    2. 消息过期后，自动转发到死信交换机
    3. 死信交换机路由到实际消费队列

### 2. 队列枚举
- **`mall-portal/src/main/java/com/macro/mall/portal/domain/QueueEnum.java`**
  - 队列名称、交换机、路由键定义

## 📝 七、日志切面（AOP）

### 1. Web日志切面
- **`mall-common/src/main/java/com/macro/mall/common/log/WebLogAspect.java`** ⭐⭐
  - **技术要点**：
    - `@Aspect`：AOP切面
    - `@Around`：环绕通知，记录请求参数、响应结果、耗时
    - 集成Logstash：使用`Markers.appendEntries()`输出结构化日志
    - 参数提取：通过反射获取`@RequestBody`、`@RequestParam`参数
  - **关键代码**：
    ```java
    @Around("webLog()")
    public Object doAround(ProceedingJoinPoint joinPoint) throws Throwable {
        long startTime = System.currentTimeMillis();
        // 记录请求信息
        WebLog webLog = new WebLog();
        Object result = joinPoint.proceed();
        // 输出到Logstash
        LOGGER.info(Markers.appendEntries(logMap), JSONUtil.parse(webLog).toString());
        return result;
    }
    ```

## 📊 九、技术架构总结

### 核心技术栈
1. **微服务框架**：Spring Cloud 2023 + Spring Cloud Alibaba
2. **API网关**：Spring Cloud Gateway（响应式）
3. **认证授权**：Sa-Token + JWT + Redis
4. **服务注册发现**：Nacos Discovery
5. **配置中心**：Nacos Config
6. **数据持久化**：MyBatis + MySQL
7. **缓存**：Redis（Jackson2Json序列化）
8. **消息队列**：RabbitMQ（延迟队列 + 死信队列）
9. **日志收集**：Logstash + ELK
10. **监控**：Spring Boot Admin

### 关键技术点

#### 1. 网关鉴权流程
```
请求 → Gateway → SaReactorFilter
  ↓
检查白名单 → 会员/管理员认证 → 动态权限校验（Redis）
  ↓
路由转发 → 后端服务
```

#### 2. 多端登录实现
- 管理员端：`StpUtil`（默认账号类型）
- 会员端：`StpMemberUtil`（自定义账号类型：`memberLogin`）
- 通过`StpInterfaceImpl`实现权限获取

#### 3. 延迟队列实现
```
发送消息 → TTL队列（延迟队列）
  ↓ (消息过期)
死信交换机 → 实际消费队列
  ↓
消费者处理
```

#### 4. Redis序列化策略
- Key：String序列化
- Value：Jackson2Json序列化（支持对象类型识别）
- 用途：缓存、Session存储、权限规则存储

#### 5. 统一异常处理
```
Controller抛出异常
  ↓
@ControllerAdvice拦截
  ↓
GlobalExceptionHandler处理
  ↓
返回CommonResult统一格式
```
